GitHub Copilot CLI vs Online

SL
Samuel Liard

Depuis quelques mois, j'utilise les deux interfaces de GitHub Copilot au quotidien. Voici un comparatif concret de ce qui fonctionne, de ce qui frustre, et du flux de travail que j'ai fini par adopter.

Le terminal change la donne

La version CLI s'intègre directement dans la boucle shell : on reste dans le contexte du projet, sans aller-retour vers le navigateur. C'est ce détail qui fait toute la différence sur les tâches répétitives.

$ gh copilot suggest "compresser le dossier logs"
# → tar -czf logs.tar.gz ./logs
Coder moins, déléguer le bruit, garder la tête pour l'architecture.

À l'inverse, l'interface web reste plus à l'aise sur les longues explications et la relecture de gros diffs, là où le terminal montre vite ses limites de lisibilité.
